# v2.4.0 Release Notes

发布日期：2026-05-28  
对应分支：`release/v2.4.0`

`v2.4.0` 是 `v2.3.1` 之后的第一个主要功能版本，主线是把 AI 患者问答从演示原型升级为可上线形态，并在此基础上完成一轮系统级的安全加固和数据完整性收口。

## 版本亮点

### 1. AI 智能体平台落地（Phase 1 → Phase 3b）

- **Phase 1 — 本地知识库**：医学知识库从 Chroma Cloud 迁到本地 pgvector，引入 `VectorBackend` / `Embedder` 抽象，默认 embedder 升级到 `BAAI/bge-m3`，新增 markdown / PDF / docx / pptx / 图片 多格式 ingest pipeline 与召回校验脚本。
- **Phase 2 — Orchestrator**：新增 `IRetriever` / `ILLMProvider`(SiliconFlow) / Tool Registry，引入 planner + executor + context + stream 四件套的 orchestrator，并把 `/api/ai/ask` 切换到新链路；同步落地三层 PII 脱敏、allowlist 校验和 audit logger。
- **Phase 3a — 同意与审计**：新增 `GET/PUT /api/profiles/me/consent` 与四档数据共享开关；orchestrator 调用前强制同意闸门、写入 `ai_prompt_audit`；新增用户侧 AI 调用历史页与同意授予/撤销历史页。
- **Phase 3b — 移动端 SSE 流式回答**：新增 `/api/ai/ask/stream` SSE 端点（含心跳与取消 plumbing），移动端 QnA 改为逐 token 渲染，支持 `[N]` 可点引用 popover、模式指示 chip、per-tool 调用 trace。

### 2. 患者档案 / 录入链路重做

- 重写 patient profile service：拆分 createProfile / updateConsent 等关键路径，引入 `SELECT FOR UPDATE` 行级锁解决多 tab 并发竞态；`patientCode` 在事务内加锁分配。
- 基线录入支持从基因报告自动 autofill（D4Z4 重复数、染色体型等），录入页 UX 同步整理。

### 3. OCR 文档类型规范化（migration 011 / 012）

- 引入 `canonicalizeDocumentType`，OCR 子类型（`muscle_mri` / `biochemistry` / `ecg` …）入库前统一映射到 4 个 canonical 值（`mri` / `genetic_report` / `blood_panel` / `other`）。
- 新增 011 / 012 两个迁移：document / status / role / severity 等 18 个 NOT VALID CHECK + 一个跨 profile 引用阻断 trigger（`patient_followup_events_same_profile`）。
- 提供完整的 down script + 历史脏数据 back-fill SQL（见 deploy runbook §2.1）。

### 4. 安全加固（PR-Sec-1 ~ PR-Sec-9）

闭合 90+ 条审计 finding，涵盖：

- **路径穿越与跨用户 FK**：文档下载 / 删除 path 收敛、`submissionId` / `linked_document_id` 跨 profile 校验。
- **PII 脱敏与 KB 注入扫描**：三层 redactor、NFKC + zero-width 预处理、prompt 注入标记告警；`scrubErrorDetail` 抽到 `ai-agents/audit/scrub.ts` 三方共享（route / orchestrator / audit reader）。
- **LLM cancel plumbing**：SSE close → AbortSignal → SiliconFlow upstream；`generateDocumentSummary` 同 path 也接上 try/finally 保证 listener 在每个 exit path 都 detach。
- **KB 服务认证**：bearer token 强制（`KB_SERVICE_HOST=0.0.0.0` 或 `NODE_ENV=production` 下缺 token 拒绝启动），body cap、where filter allowlist、`do_POST` 改 default-deny。
- **解析器 DoS 防护**：Pillow image bomb、docx zip bomb、YAML 大小上限。
- **生产 env fail-fast**：`JWT_SECRET` / `OTP_HASH_SECRET` / `OTP_PROVIDER` / `CORS_ORIGIN` / `OCR_PROVIDER` / `KB_SERVICE_TOKEN` 缺值、`change-me-*` 或 `dev-only-*-NOT-FOR-PROD` 占位符时拒绝启动。
- **依赖漏洞收口**：`npm audit fix` 消除 API 8 高危（`multer` DoS、`jws`、`lodash`、`path-to-regexp` 等）+ Mobile 9 高 1 critical，剩余 moderate 全为 build-time 传递依赖。

### 5. 移动端体验与可靠性

- Logout 清理 chat cache、401 自动 logout、AppState 切后台时取消 SSE 流。
- 数据共享开关串到后端、AI 调用历史 + 同意历史可视化、引用 snippet cap、newly registered account 404 路由到 onboarding。
- `AssistantMetadata` 类型与组件名解构、`API_BASE_URL` 单点 export。

### 6. 部署 / 运维材料

- 新增 `docs/runbooks/v2.4.0-deploy.md`：覆盖 env 准备、KB 镜像、Postgres extension、staging 验证 checklist、prod 上线顺序、监控基线、回滚 SQL。
- 新增 `scripts/smoke-test.sh` v2.4.0 头条覆盖块：重复注册 409 / `document_type` canonicalize / sharing prefs 读写 / AI consent + audit / 跨用户 submission 404。
