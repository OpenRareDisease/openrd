# v2.3.1 Release Notes

发布日期：2026-03-31  
对应分支：`release/v2.3.1`

`v2.3.1` 是在 `v2.3.0` 基础上的发布收口版本，重点不是新增页面，而是把部署路径、OCR 运行路径和文档入口统一到当前真实实现，去掉已经不用的旧兼容层。

## 版本亮点

### 1. OCR 主链路收敛为 embedded 模式

- API 只保留 `embedded` / `baidu` / `mock` 三种 OCR provider。
- 删除旧的 `report_manager` HTTP provider 兼容代码与轮询刷新分支。
- `apps/report-manager` 目录只保留嵌入式 parser、OCR 服务和 FSHD 结构化解析代码。

### 2. 旧独立服务残留清理

- 删除 `report-manager` 的 FastAPI 入口、Dockerfile、数据库初始化脚本、MinIO 初始化脚本和旧上传测试脚本。
- 删除不再生效的 `REPORT_MANAGER_OCR_*`、`REPORT_MANAGER_MINIO_*` 等旧环境变量入口。
- 同步收紧 API Dockerfile 和本地 `.env`，避免继续暴露已经无效的配置项。

### 3. 文档与版本信息整理

- 将设计提案、历史文档和当前事实文档分层整理。
- 删除 `CLAUDE.md`，保留面向仓库使用者的正式文档入口。
- 将当前代码版本统一更新到 `2.3.1`，补齐 changelog 与 release note。
